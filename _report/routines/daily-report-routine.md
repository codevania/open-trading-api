# 일일 리포트 생성 루틴

## 루틴 이름

일일 투자 리포트 생성

## 실행 트리거

사용자가 다음과 같이 요청하면 이 루틴을 실행한다.

- "오늘 리포트 뽑아줘"
- "데일리 리포트 작성"
- "일일 리포트 만들어줘"
- "루틴 돌려줘"
- "장마감 리포트"

## 실행 시점

- 기본 기준일: 현재 한국시간 날짜
- 기본 작성 시점: 한국장 마감 이후
- 해외주식은 전일 또는 현재 확인 가능한 NASDAQ 데이터를 함께 기록한다.
- 휴장일이거나 데이터가 비정상적이면 리포트에 `데이터 품질 메모`로 남긴다.

## 입력 파일

- 기본 관찰종목 예시: `_report/watchlist.example.yaml`
- 개인 관찰종목: `_report/watchlist.yaml`
- 리포트 템플릿: `_report/templates/daily-report.md`
- 결정 로그: `_report/decisions/decision-log.md`

`_report/watchlist.yaml`이 있으면 우선 사용한다. 없으면 `_report/watchlist.example.yaml`을 기준으로 삼는다.

## 기본 관찰종목

현재 루틴의 기본 관찰종목은 다음과 같다.

| 종목코드 | 종목명 | 시장 | 상태 |
| --- | --- | --- | --- |
| 005930 | 삼성전자 | KRX | 관찰중 |
| 000660 | SK하이닉스 | KRX | 관찰중 |
| NAS:NVDA | NVIDIA | NASDAQ | 관찰중 |
| 확인필요 | 현대 로보틱스 | KRX | 종목 확인 필요 |

종목이 추가되면 `_report/watchlist.yaml`에 먼저 반영하고, 리포트의 `관찰종목` 메타데이터에도 포함한다.

## MCP 조회 순서

국내주식은 API 호출 전에 반드시 `find_api_detail`로 파라미터를 확인한다.

1. 종목코드 확인
   - `domestic_stock.find_stock_code`
2. 현재가
   - `domestic_stock.inquire_price`
3. 일봉
   - `domestic_stock.inquire_daily_itemchartprice`
   - 기본 조회 범위: 기준일 이전 60일
   - 리포트에는 최근 10거래일 내외의 종가 그래프를 포함한다.
4. 투자자 수급
   - `domestic_stock.investor_trade_by_stock_daily`
   - 수급이 판단 근거일 때는 최근 외국인 또는 기관 순매수 그래프를 함께 포함한다.
5. 뉴스/공시 제목
   - `domestic_stock.news_title`
   - 표준 파라미터가 확정되지 않았으면 무리하게 호출하지 말고 `미수집`으로 기록한다.

해외주식은 다음 API부터 사용한다.

1. 현재체결가
   - `overseas_stock.price`
2. 기간별시세
   - `overseas_stock.dailyprice`

## 리포트 작성 절차

1. 기준일을 확정한다.
2. 관찰종목 목록을 확인한다.
3. 각 API의 상세 파라미터를 확인한다.
4. 종목별 데이터를 조회한다.
5. 종목별 가격·거래량·수급 중 판단에 중요한 항목을 그래프로 만든다.
6. `_report/daily/YYYY/YYYY-MM-DD.md`를 작성한다.
7. 종목별 판단을 `_report/decisions/decision-log.md`에 추가한다.
8. 누락 데이터, API 실패, 종목명 불확실성을 `데이터 품질 메모`에 적는다.
9. 사용자가 요청하지 않는 한 주문 API는 호출하지 않는다.

## 그래프 작성 규칙

- 기본 형식은 Markdown Mermaid `xychart-beta`를 사용한다.
- 각 정상 관찰종목에는 최소 1개 이상의 가격 흐름 그래프를 둔다.
- 가격만으로 판단하기 어려운 국내 종목은 외국인 또는 기관 순매수 그래프를 추가한다.
- 정확한 종목코드가 확정되지 않은 항목은 확정 종목처럼 차트를 만들지 말고 후보 비교 그래프로 표시한다.
- 그래프 아래에는 반드시 그 그래프를 참조한 해석을 1개 이상 적는다.
- 그래프 단위는 제목 또는 y축에 명확히 적는다. 예: 원, 천원, 만원, 달러, 백만 주, 지수.

## 판단 표현 규칙

판단값은 한국어로 통일한다.

- 관망
- 매수 후보
- 추가 확인
- 비중 축소 후보
- 매도 후보
- 종목 확인

각 판단에는 반드시 다음을 붙인다.

- 근거
- 실행 조건
- 무효화 조건
- 재검토일

## 저장 규칙

- 사람이 읽는 리포트: `_report/daily/YYYY/YYYY-MM-DD.md`
- 결정 누적: `_report/decisions/decision-log.md`
- 원천 JSON: `_report/raw/YYYY/YYYY-MM-DD/SYMBOL/`

현재는 수동 루틴에서 원천 JSON을 기본 저장하지 않는다. 저장하지 않은 경우 리포트에 `조회 완료, 파일 미저장`으로 남긴다.

## 완료 기준

루틴은 다음이 완료되면 끝난다.

- 기준일 일일 리포트가 생성 또는 갱신됨
- 관찰종목별 그래프와 그래프 기반 해석이 있음
- 관찰종목별 판단이 있음
- 결정 로그가 갱신됨
- 누락 데이터와 확인 필요 항목이 명시됨
- 실제 주문 API를 호출하지 않았음
