## 9. 데이터 품질 메모

- 입력 파일: 루트 [[_report/watchlist.yaml|_report/watchlist.yaml]]과 `_report/routines/daily-report-routine.md`가 없어 [[_report/di/watchlist.yaml|_report/di/watchlist.yaml]]과 [[_report/di/routines/daily-report-routine|_report/di/routines/daily-report-routine.md]]를 사용했다. 사용자가 SK스퀘어, SKC, LG이노텍, 삼성전기를 추가 요청해 [[_report/di/watchlist.yaml|_report/di/watchlist.yaml]]에 반영하고 같은 기준일 리포트에 편입했다. 산출물은 사용자 요청대로 루트 [[_report/daily/2026/2026-07-01|_report/daily/2026/2026-07-01.md]]와 [[_report/decisions/decision-log|_report/decisions/decision-log.md]]에 저장한다.
- KIS preflight: 현재 Codex App 표면에는 직접 KIS MCP 도구가 노출되지 않아, repo-local MCP `configs/*.json`의 `find_api_detail` 동등 정보를 저장한 뒤 checked-in `examples_llm` 조회성 래퍼만 호출했다.
- 국내 투자자 수급: `investor_trade_by_stock_daily`는 `OPSQ2001 / TIME LIMIT 00:00 ~ 15:40`로 응답해 output이 비었다. 정식 투자자 수급은 미확인이다.
- 현재가 응답 수급성 필드: `frgn_ntby_qty`, `pgtr_ntby_qty`는 보조 참고값으로만 사용했다. 기관 수급이나 누적 투자자 수급으로 확장 해석하지 않았다.
- 해외 `dailyprice`: pagination 결과가 200행이지만 날짜 기준 중복 제거 후 100개 거래일로 해석했다.
- 뉴스/공시: 이번 요청의 필수 범위가 현재가/기간별 시세/수급이어서 실제 뉴스 API는 호출하지 않았다.
- 실제 주문 API: 호출하지 않았다.
