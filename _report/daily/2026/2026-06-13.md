## 9. 데이터 품질 메모

- 사용자가 요청한 `_report/routines/daily-report-routine.md`, [[_report/watchlist.yaml|_report/watchlist.yaml]], `_report/watchlist.example.yaml`은 현재 프로젝트에 없었다. 실제 존재하는 [[_report/di/routines/daily-report-routine|_report/di/routines/daily-report-routine.md]]와 [[_report/di/watchlist.yaml|_report/di/watchlist.yaml]]을 기준으로 수행했다.
- 사용자가 명시한 결과 경로를 우선해 리포트는 [[_report/daily/2026/2026-06-13|_report/daily/2026/2026-06-13.md]], 결정 로그는 [[_report/decisions/decision-log|_report/decisions/decision-log.md]]에 작성했다.
- 2026-06-13은 토요일이다. 국내 최신 거래일은 KIS 응답 기준 2026-06-12다.
- KIS MCP API 호출 전 `find_api_detail`로 국내 종목검색, 현재가, 기간별시세, 투자자 수급, 뉴스 제목, 해외 현재체결가, 해외 기간별시세 파라미터를 확인했다.
- 국내 `investor_trade_by_stock_daily`는 `OPSQ2001 / TIME LIMIT 00:00 ~ 15:40` 응답으로 빈 배열을 반환했다. 따라서 별도 투자자매매동향은 미확인이다.
- 현재가 응답의 `frgn_ntby_qty`, `pgtr_ntby_qty`는 참고용으로만 사용했다. 장마감 투자자 수급을 대체하는 확정 근거로 보지 않는다.
- `news_title`은 상세 파라미터를 확인했지만 뉴스 제공업체 코드, 시장 구분 코드, 정렬 코드의 문서화된 값을 확정하지 못해 호출하지 않았다.
- PowerShell heredoc 경로에서 한글 종목명 검색 입력이 깨져, `find_stock_code`는 ASCII 종목코드로 재확인했다. 세 국내 종목 모두 코드 기준 exact match가 확인됐다.
- 실제 보유수량, 평균단가, 목표비중, 계좌 위험도는 미기록이다.
- 실제 주문 API는 호출하지 않았다.
