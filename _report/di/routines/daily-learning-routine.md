# DI Daily Learning Routine

## 목적

재량 투자 리서치를 매일 학습할 수 있도록 `오늘의 학습 콘텐츠`를 만들고 `_report/di/learning/logs/YYYY-MM.md`에 누적한다. 이 루틴은 공부와 기록을 위한 것이며, 매수/매도/보유 판단이나 주문 의도를 만들지 않는다.

## 트리거 문구

사용자가 다음처럼 요청하면 이 루틴을 따른다.

- "오늘 학습"
- "오늘 공부할 내용"
- "DI 학습 루틴"
- "투자 공부 루틴"
- "매일 학습 내용 만들어줘"
- "학습 일지 이어서 써줘"

기본 입력:

- [[_report/di/learning/daily-log-template|daily-log-template.md]]

- [[_report/di/README|_report/di/README.md]]

## Day 결정 규칙

1. 날짜는 기본적으로 한국시간(Asia/Seoul)의 `YYYY-MM-DD`를 사용한다.
2. 사용자가 특정 Day를 지정하면 그 Day를 사용한다.
3. 지정이 없으면 월별 로그에서 마지막 `Day N`을 찾고 다음 숫자를 사용한다.
4. `Day 0 Setup`은 학습 Day 계산에서 제외한다.
5. 로그가 없거나 Day가 없으면 `Day 1`부터 시작한다.
6. 며칠 쉬었더라도 달력 일수로 건너뛰지 않는다. 마지막 완료 Day의 다음 Day를 이어간다.
7. `Day 56` 이후에는 `Review N`으로 전환하고, 이전 로그의 미해결 질문과 evidence gap을 복습 주제로 삼는다.

## 기본 출력 위치

```text
_report/di/learning/logs/YYYY-MM.md
```

해당 월 파일이 없으면 새로 만든다.

## 중복 방지

같은 날짜의 항목이 이미 있으면 새 `## YYYY-MM-DD - Day N` 제목을 추가하지 않는다. 기존 날짜 항목 아래에 `오늘의 학습 콘텐츠`가 없으면 추가하고, 이미 있으면 `업데이트` 하위 항목으로 덧붙인다.

## 콘텐츠 생성 원칙

- [[_report/di/learning/learning-plan-2026-07-08|learning-plan-2026-07-08.md]]의 해당 Day 주제를 기본으로 삼는다.
- 초보자에게 바로 필요한 설명만 쓴다. 어려운 용어는 한 문장으로 풀어쓴다.
- 하루 산출물은 작게 둔다: 핵심 개념 1개, 읽을 자료 1~2개, 직접 쓸 문장 3~5개.
- ETF 주제는 비용, 유동성, NAV/괴리, 분배, 세금/계좌 확인 질문으로 연결한다.
- 기업 주제는 사업 모델, 재무제표, 공시, valuation, 무효화 조건으로 연결한다.
- 후보를 다룰 때는 "살 만한가"보다 "무엇이 아직 미확인인가"를 먼저 쓴다.
- 세금, 수수료, 상품 상세, 최신 공시처럼 바뀔 수 있는 정보는 공식 출처 확인이 필요하다고 표시한다.
- 공식 출처를 새로 확인했다면 링크와 확인일을 남긴다.
- 웹 검색이 필요하면 공식/원천 출처를 우선한다: SEC, DART, KRX, 운용사, 거래소, 감독기관, 회사 IR.

## 금지 사항

- 매수/매도/보유 추천처럼 쓰지 않는다.
- 실제 주문, 주문 가능 API, 계좌번호, 보유수량, 앱키, 토큰, `.env` 내용을 기록하지 않는다.
- AI 답변만으로 사실을 확정하지 않는다.
- 커뮤니티, 블로그, 위키를 판단 근거로 사용하지 않는다.
- 같은 날짜 항목을 중복 생성하지 않는다.

## 완료 확인

루틴을 실행한 뒤 다음을 확인한다.

1. 오늘 날짜 항목이 월별 로그에 존재한다.
2. `오늘의 학습 콘텐츠`가 포함되어 있다.
3. 참고 자료와 연결 파일이 적혀 있다.
4. 투자 결론이 아니라 판단 보류 메모가 있다.
5. 민감정보가 들어가지 않았다.
